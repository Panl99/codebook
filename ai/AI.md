


# AI组件

## Agent

智能体（人工智能代理），是一个能够感知环境、自主决策、执行动作以实现特定目标的智能系统。

智能体像人一样，它有记忆、有逻辑分析能力、有任务的拆解能力、问题的拆解能力和最后综合回来统一解决问题的能力。它不再是简单地被动回答问题，而是主动地去完成任务。

**核心特征：与普通AI（如ChatGPT）的区别**

特性|	普通AI模型（如ChatGPT）|	AI Agent（智能体）
---|---|---
工作模式|	被动响应：你问，它答。一次交互，一次回复。|	主动代理：你给定一个目标，它自己规划、执行、调整，直到完成。
交互方式|	单轮或有限轮对话。|	多轮、闭环。与环境持续交互，根据反馈调整行动。
核心能力|	内容生成、信息整合、对话。|	自主决策、任务分解、工具使用、学习与适应。
比喻|	一个博学且反应迅速的百科全书或顾问。|	一个能独立完成项目的实习生或执行者。


**AI Agent 的核心组成部分（架构）**

一个典型的AI Agent通常包含以下几个模块：

1. **规划模块**
    - 任务分解：将复杂目标拆解为可执行的子任务序列。（例如，目标“写一份行业报告” → 拆解为：搜索资料、分析数据、撰写大纲、完成初稿、润色修改）。
    - 战略制定：决定完成任务的最佳路径和策略。
2. **记忆模块**
    - 短期记忆：记录当前任务链的上下文和中间状态。
    - 长期记忆：存储过去的经验、学到的知识、用户偏好等，用于未来任务，实现持续学习和个性化。
3. **工具使用模块**
    - 这是Agent能力扩展的关键。它不仅可以“思考”，还能调用各种外部工具和API来行动，例如：
        - 搜索工具：上网查询最新信息。
        - 代码执行器：运行代码分析数据。
        - 软件操作：控制浏览器、操作文档（如Word/Excel）、发送邮件。
        - 专属系统：连接企业内部的数据库、CRM等。
4. **行动与执行模块**
    - 根据规划，调用相应的工具或自身能力，执行具体动作，并观察结果。
5. **反思与学习模块（高级）**
    - 评估行动结果是否有效。
    - 从失败中学习，调整后续计划。
    - 这是实现持续进化和强适应性的关键。


**AI Agent 的工作流程（以“订一张最便宜的去上海的机票”为例）**

1. 感知目标：接收用户指令：“帮我订一张下周五从北京飞上海的最便宜的机票。”
2. 规划：
    - 子任务1：搜索下周五北京到上海的所有航班。
    - 子任务2：比较价格，找出最便宜的选项。
    - 子任务3：查询该航班的退改签政策。
    - 子任务4：为用户预订该航班。
3. 执行与工具调用：
    - 调用搜索工具/爬虫访问航旅网站，获取航班列表和价格（任务1）。
    - 使用代码/逻辑对比价格，筛选出最便宜的（任务2）。
    - 再次调用搜索工具查看该航班的详细规则（任务3）。
    - 调用预订API或模拟填写表单，完成下单（任务4）。过程中可能需要向用户确认身份信息。
4. 反馈与迭代：
    - 如果预订失败（如支付问题），它会反思原因，并尝试备用方案（如换支付方式或选择次优航班）。
5. 输出结果：将预订成功的确认信息（航班号、时间、价格等）反馈给用户。


**AI Agent 的类型**

- 按自动化程度：
    - 全自动Agent：无需人工干预，全程自主运行。
    - 半自动Agent：在关键节点需要人工确认或批准。
- 按应用领域：
    - 个人助理Agent：管理日程、安排旅行、处理邮件。
    - 创作与研究Agent：自动撰写报告、分析文献、生成代码。
    - 客服与销售Agent：全流程处理客户咨询、订单、售后。
    - 游戏与仿真Agent：在复杂环境中扮演有策略的NPC。
    - 科研Agent：自主设计实验、分析数据、提出假设。


**挑战与局限性**

- 可靠性问题：复杂的任务链中，一个环节出错可能导致整个任务失败（“幻觉”问题在长链条中被放大）。
- 安全性风险：自主执行动作可能带来未经授权的操作（如误发邮件、错误修改数据）。
- 效率与成本：反复调用大模型和工具进行“思考”，耗时且昂贵。
- 伦理与责任：当Agent出错造成损失时，责任归属不清晰。

## MCP

模型上下文协议（Model Context Protocol）

Anthropic提出的一种开放标准协议，用于统一AI模型（如大语言模型）安全、动态地与外部数据源和工具进行交互。

核心作用：解决“数据孤岛”问题。以往AI模型若想查数据库、发邮件或操作本地文件，需要开发者编写大量复杂且不兼容的代码。MCP遵循客户端-服务器架构，充当了AI与外部世界之间的“通用USB-C接口”。

核心概念组件：
- MCP Hosts: AI 应用程序(如 Claude Desktop、IDE)
- MCP Clients: 应用内的协议客户端
- MCP Servers: 提供特定功能的服务(如访问数据库、文件系统、API)
- 本地数据源: 你的业务数据、内容库等
- 远程服务: 通过互联网访问的 API 和服务

应用场景：让AI助手自动连接你的本地数据库、GitHub仓库或Slack工作区，实现真正的“操作执行”而不仅是“文本生成”。



## RAG

检索增强生成（Retrieval-Augmented Generation）

一种通过实时检索外部知识库来“临时”增强大模型回答质量的技术架构。

核心作用：解决大模型的“幻觉”和知识滞后问题。RAG不修改模型本身的参数，而是在用户提问时，先去指定的知识库（如公司内部文档、最新网页）中检索相关信息，将这些信息作为“参考资料”连同问题一起发给模型，让模型基于事实进行总结。

工作原理:
1. 检索 (Retrieval): 根据用户的问题,从知识库中搜索相关文档或信息片段
2. 增强 (Augmentation): 将检索到的信息与用户问题组合,形成增强后的提示词
3. 生成 (Generation): 大语言模型基于增强后的上下文生成回答

应用场景：企业知识库问答、智能客服（确保答案基于最新手册而非过时的训练数据）。



## Skills

技能

在智能体框架中，为AI封装好的、可复用且可调用的“能力单元”。

核心作用：将复杂的多步骤任务抽象为单一指令。如果将AI比作一个操作系统，Skills就是安装在上面的“应用程序”或“插件”。相比于MCP侧重“连接底层资源”，Skills更侧重“封装高层逻辑”。例如，一个“数据分析Skill”可能内部包含了“连接数据库、编写Python代码、生成图表、发送邮件”等多个步骤，用户只需说“运行数据分析Skill”即可。

应用场景：在Claude、GPTs或各类AI Agent平台中，通过定制Skills让AI完成专业的代码审查、特定格式的报告撰写或复杂的跨系统工作流。

‌典型结构‌（以 Claude Skills 为例）：
- SKILL.md：包含 YAML 元数据（名称、描述、允许工具）和 Markdown 工作流。
  - 公共 Skills (/mnt/skills/public/): 如创建 Word 文档、PowerPoint、Excel 等
  - 示例 Skills (/mnt/skills/examples/): 演示性质的技能
  - 用户 Skills (/mnt/skills/user/): 用户自定义上传的技能
- scripts/：可选执行脚本（如 Python、Shell）。
- references/：参考文档（按需加载）。




## AGI

通用人工智能（Artificial General Intelligence）

是指具有高效的学习和泛化能力、能够根据所处的复杂动态环境自主产生并完成任务的通用人工智能体，具备自主的感知、认知、决策、学习、执行和社会协作等能力，且符合人类情感、伦理与道德观念。



# AI Coding


## Vibe Coding

Vibe Coding（氛围编程）是一种由人工智能辅助的编程方式，其核心特点是开发者只需通过自然语言描述需求（动口），由AI生成并执行代码（动手），从而大幅降低编程的技术门槛。‌‌
