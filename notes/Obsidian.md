
# Obsidian

Obsidian 的核心在于利用本地 Markdown 文件构建知识网络，结合插件实现日程规划、图片管理及卡片笔记法等高效工作流。

- 本地文件：所有笔记都是 Markdown 文件，存本地文件夹（叫“Vault”），随时可以用 Git 同步或备份。
- 双向链接：像维基百科一样，笔记可以互相链接，形成知识网络。
- 插件生态：上千个社区插件，能扩展成任务管理器、数据库、绘图工具等。
- 适合开发者：完美支持代码高亮、Git 集成、查询笔记如数据库。

## 操作

- 使用Markdown编写
- 双向链接：输入 `[[笔记名]]` 就会创建链接。如果笔记不存在，点击会自动新建。
  - 示例：在 “Python Tips” 笔记里写 [[FastAPI 入门]]，以后就能互相跳转。
- 标签（Tags）：用 `#tag`，比如 #python #todo，便于搜索。
- 图谱视图（Graph View）：左侧栏点击“图谱”图标，能看到所有笔记的连接关系，越写越多越像大脑网络。

## 搭建笔记结构

1. 采用 PARA 方法分类：避免文件夹嵌套过深，推荐建立五大文件夹，配合标签和双向链接组织内容。
   - Inbox（临时收集，临时捕捉的想法、链接）
   - Projects（当前项目，正在进行中的任务）
   - Areas（责任领域，如 “Frontend”、“Backend”、“DevOps”）
   - Resources（常青知识，如 “Python”、 “React”、 “算法”）
   - Archives（归档，已结束的任务）
   - 
   - Daily：自动生成的日记（Daily Notes 会放这里）。
   - Snippets：代码片段库，按语言分文件夹。
     - java
     - python

   不要过度嵌套文件夹，靠链接和标签组织更灵活。
2. 统一附件管理路径：使用 Custom Attachment Location 插件设置附件自动归集到`./${filename}_assets` 文件夹，重命名笔记时附件文件夹同步更新，定期清理游离文件 。
3. 标准化文件命名：建议采用`YYYY-MM-DD` 标题 或` 项目 - 主题` 格式，便于排序和检索，所有笔记默认保存为 `.md` 格式以兼容 Git 等同步工具 。

## 必装插件与工作流

1. 核心效率插件：
   - `Daily Notes`：每天自动生成日记，适合记录学习心得。
   - `Graph View`：知识图谱。
   - `Outline`：显示当前笔记大纲。
   - `Backlinks`：显示哪些笔记链接到当前笔记。
   - `Page Preview`：鼠标悬停链接时预览内容。
2. 社区插件推荐（开发者必备）  
   设置 → 第三方插件 → 关闭安全模式 → 浏览。
   - `Calendar`：日历视图，配合 `Daily Notes` 使用，点击日期快速创建或跳转日记，记录每日进度 。
   - `Dataview`：把笔记当数据库查询，自动聚合带特定标签(`#todo`)的任务或笔记，生成动态表格。
   - `Templater`：自动插入模板，新建项目或日记时自动填充日期、状态和固定结构 。
   - `Tasks`：强大任务管理，管理待办事项，支持截止日期、优先级和重复任务，可查询未完成任务并生成清单 。
   - `Obsidian Git`：直接在 Obsidian 里 Git commit/push/pull，推荐配合 GitHub 私有仓库使用，设置自动 commit（如每 5 分钟），实现多设备同步和版本历史管理 。
   - `Excalidraw`：在笔记中直接绘制架构图或流程图，支持双向链接嵌入，适合产品经理和设计师
   - `Advanced Tables`：像 Excel 一样编辑表格。
   - `Linter`：自动格式化笔记（统一标题、空格等）。
   - `Code Block Enhancer` 或 `Better CodeBlock`：增强代码块功能（复制按钮、行号等）。
   - `Omnisearch` + `TextExtractor`：实现全文检索，支持 PDF 和图片内的文字识别，检索效率比原生搜索更高 。
   - 其他可选：QuickAdd（快速创建笔记）、Homepage（自定义首页）。

## 场景化应用技巧

1. 卡片笔记法：
   - 控制卡片大小：每张卡片建议 100-200 字，便于复用和组合，避免单张卡片过长导致难以嵌入文章 。
   - 建立知识对撞机：定期将不同领域的卡片进行关联思考，如将“番茄工作法”与“知行合一”结合，产生新观点 。
2. 任务与项目管理：
   - 每日 TODO 工作流：使用 Daily TODO 插件创建独立的日期任务文件，自动将未完成任务滚动到次日，与日记分离管理 。
   - 外部工具集成：若需更强任务管理，可集成 Google Tasks，在 Obsidian 中链接任务列表，实现轻量级捕获与专业管理结合 。
3. 学习与写作闭环：
   - 记录—关联—复盘：学习时快速记录碎片，定期通过双向链接关联旧知识，复习时利用图谱定位薄弱环节 。
   - 输出深度知识：写作时直接调用沉淀的卡片和素材，将零散笔记转化为文章或报告，避免重复劳动 。


笔记组织技巧
- MOC（Map of Content）：每个大主题创建一个“地图”笔记，列出相关子笔记链接。 
  - 示例：创建 “Python MOC.md”，里面列出 [[Python 基础]]、[[FastAPI]]、[[数据分析]] 等。
- 原子笔记：每篇笔记只写一个主题，便于链接复用。
- 属性（Properties）：在新版 Obsidian 中，笔记开头用 YAML 添加元数据：
   ```
   ---
   tags: [python, web]
   status: in-progress
   created: 2026-01-31
   ---
   ```
   配合 Dataview 查询超级强大。

代码与技术笔记
- 用代码块写片段，指定语言高亮。
- 创建 “CodeSnippets” 文件夹，按语言/框架分类。
- 用 Dataview 查询：列出所有 Python 相关的笔记或代码片段。
- 画图：Excalidraw 画系统架构、数据库 ER 图。

项目与任务管理
- 每个项目一个 MOC 笔记，链接需求文档、API 设计、会议记录。
- 用 Tasks 插件管理 TODO，支持查询未完成任务。
- Daily Notes 记录每天进度、问题、解决方案。

同步与备份（重要⭐）
- 推荐方案：用 Obsidian Git 插件 + GitHub/GitLab 私有仓库。
  - 安装后设置自动 commit（每5分钟）。
  - 多设备同步完美，且有版本历史。
- 备选：Syncthing（免费跨平台同步）、Obsidian Sync（官方付费，加密）。

进阶技巧
- 模板：用 Templater 创建项目模板、新笔记模板。
- 查询示例（Dataview）：
   ```
   TASK
   WHERE !completed AND due <= date(today) + dur(7 days)
   ```
  （列出7天内到期任务）
- 自定义 CSS：设置 → 外观 → CSS snippets，自定义代码块样式、主题。
- 从简单开始：先用纯 Obsidian 写几周，再逐步加插件。不要一开始就追求完美。


# Obsidian对比Notion

- `Obsidian`：本地优先的**个人知识管理**工具，所有笔记以标准Markdown格式存储在本地设备，主打数据完全自主可控，通过双向链接构建非线性的个人知识网络，核心服务于个人深度思考与知识沉淀。
- `Notion`：云端原生的**一体化协作**生产力平台，以块和数据库为核心架构，整合了文档、项目看板、知识库等多种功能，主打开箱即用的结构化工作流，核心服务于团队协同与标准化任务管理。

| 对比维度    | Obsidian                | Notion                |
|---------|-------------------------|-----------------------|
| 数据存储    | 本地纯Markdown存储，完全离线可用    | 云端托管，离线功能受限           |
| 协作能力    | 原生单用户设计，共享 vault 体验一般   | 支持多人实时编辑、权限管控，协作流畅    |
| 上手门槛    | 需掌握基础Markdown，插件配置有学习成本 | 开箱即用，大量现成模板可直接套用      |
| AI能力    | 依赖社区插件实现，自由度高但需自行配置     | 内置Notion AI，深度整合全平台功能 |
| 性能表现    | 本地读取速度极快，大库无明显卡顿        | 大数据库依赖网络，复杂页面易出现延迟    |
| 数据安全    | 完全由用户掌控，无厂商锁定风险         | 数据托管在官方服务器，导出流程繁琐     |


适用场景选择建议
- 优先选`Obsidian`：独立研究者、写作者、开发者，重视数据隐私，需要深度关联零散笔记构建个人知识体系，愿意花少量时间自定义工具工作流。
- 优先选`Notion`：和团队共同完成项目管理、共享知识库，偏好无需配置的一体化体验，日常以结构化任务和协同文档为核心使用场景。
- 搭配使用：用`Obsidian`记录零散灵感、关联深度笔记，完成内容沉淀后再迁移到`Notion`中做结构化的项目追踪和团队共享，实现两款工具的优势互补。
